# 🧠 如何使用這本攻略 (User Guide)

這不是一般的教科書。這是為了**你的大腦**設計的操作手冊。
針對 ADHD/GAD 的特質（工作記憶有限、容易焦慮癱瘓），我們採用 **「認知卸載 (Cognitive Offloading)」** 策略。

簡單說：**不要試圖在腦中執行程式碼。把思考過程「外包」給紙筆或註解。**

---

## 🛠️ 解題標準作業程序 (SOP)

遇到任何題目，請嚴格遵守這個 SOP。這能救你在腦袋一片空白的時候。

### 1. 閱讀與翻譯 (Read & Translate)
*   讀題目，然後用**你自己的話**，在程式碼最上方寫下這題要在幹嘛。
*   **例子**：
    ```python
    # 題目：給你一堆數字，找出有沒有重複的。
    # 輸入：[1, 2, 3, 1] -> 輸出：True
    # 輸入：[1, 2, 3, 4] -> 輸出：False
    ```

### 2. 畫圖與舉例 (Draw & Example)
*   **絕對不要**直接開始寫 code。
*   在紙上，或是在註解裡，畫出這題的樣子。
*   用一個簡單的例子，手動跑一遍流程。
    *   *「如果我是電腦，我會先看 1，記下來... 再看 2...」*

### 3. 微步驟計畫 (Micro-Planning)
*   這是最重要的。把要做的每一件小事，列成清單 (Todo List)。
*   **這就是把「焦慮的大怪獸」切成「無害的小史萊姆」。**
*   **寫成註解**：
    ```python
    def containsDuplicate(nums):
        # 1. 建立一個空的HashSet來存看過的數字
        # 2. 寫一個迴圈，一個一個看nums裡面的數字
        #    a. 如果這個數字已經在Set裡面 -> 代表重複了 -> 回傳 True
        #    b. 如果不在Set裡面 -> 把它加進Set
        # 3. 如果迴圈跑完了都沒發現重複 -> 回傳 False
        pass
    ```
*   **看到沒？程式碼自然而然就出來了。你只需要翻譯這些中文。**

### 4. 實作與驗證 (Implement & Verify)
*   照著你的註解微步驟，一行一行翻譯成 Python (或你用的語言)。
*   寫完後，回頭看你的「例子」，在腦中或紙上跑一遍。

## 🚑 緊急狀況處理

### Q: 我完全沒頭緒，怎麼辦？
**A:** 給自己 **15 分鐘**。真的想不出來，**立刻看解答**。
*   這不是作弊，這是「學習」。
*   但請只看**思路 (Idea)**，然後試著自己寫出微步驟。
*   如果不看 Code 還是寫不出來，那就看 Code，然後**抄一遍**。抄寫能幫助大腦建立肌肉記憶。

### Q: 我覺得我很笨，這麼簡單都不會。
**A:** 停！那是 GAD 在騙你。
*   演算法是「模式 (Pattern)」。不會是因為你還沒看過這個模式，不是因為你笨。
*   就像玩遊戲不知道密技一樣，去找攻略就好。

### Q: 我怕我學了又忘。
**A:** 這是為了這本書存在的目的。
*   我們會用 **「間隔重複 (Spaced Repetition)」** 的概念。
*   你會發現前面的題目概念會在後面重複出現。
*   而且，忘記了就回來查，這就是為什麼我們要把所有東西都**寫下來**。
